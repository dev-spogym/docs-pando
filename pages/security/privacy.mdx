## 수집 항목

개인정보보호법 제16조는 필수 항목과 선택 항목을 명확히 구분하여 동의를 받도록 의무화하고 있습니다. 선택 항목을 필수처럼 포장하거나 동의를 강요하면 법 위반입니다. 건강 상태·체성분 등 민감 정보는 별도 동의가 필요하며, 금융 정보는 전자금융거래법의 추가 보호 의무가 적용됩니다.

## 민감 정보 마스킹

마스킹 정책은 **최소 권한 원칙(Principle of Least Privilege)**에 기반합니다. 업무 수행에 필요한 수준 이상의 정보를 노출하지 않음으로써, 내부 직원에 의한 정보 오남용 위험을 줄입니다. 예를 들어 스태프가 회원에게 전화를 걸기 위해서는 번호 마지막 4자리만으로도 본인 확인이 가능하므로, 전체 번호를 보여줄 이유가 없습니다.

### CRM 화면 마스킹

- **마스킹 적용**: CRM 화면에서 역할에 따라 자동 마스킹 (스태프는 연락처 뒷 4자리만 표시)
- **원본 열람**: 지점장 이상 역할만 마스킹 해제 버튼 표시

| 정보 | 마스킹 | 원본 열람 |
|---|---|---|
| 전화번호 | 010-\*\*\*\*-5678 | 지점장+ |
| 주민번호 | 뒷자리 전체 | 본사(회장)만 |
| 계좌번호 | 뒤 4자리만 | 본사+ |
| 직원 급여 | 본인만 전체 | 본사 (전체) |

### 엑셀 다운로드 시

엑셀 파일 다운로드는 데이터를 시스템 외부로 반출하는 행위입니다. 화면에서는 마스킹이 적용되어 있더라도, 파일로 저장되는 순간 통제를 벗어납니다. 따라서 다운로드 파일에도 역할별 마스킹을 동일하게 적용하여, 파일 유출 시에도 피해를 최소화합니다.

- **엑셀 다운로드**: 역할별 마스킹이 적용된 상태로 다운로드됩니다

| 역할 | 마스킹 |
|---|---|
| 회장/대표 | 해제 |
| 본사 직원 | 주민번호만 |
| 지점장 | 전체 마스킹 |

### 개인정보 파기

- **휴면 전환 시**: 시스템이 자동으로 개인정보 파기 처리 (민감 정보 즉시 파기)
- **수동 파기 요청**: 회원 탈퇴 요청 또는 별도 파기 요청 시 지점장 → 본사 결재 후 처리

## 백업/복구

백업 정책은 두 가지 핵심 지표로 설계됩니다. **RPO(Recovery Point Objective, 복구 목표 시점)**는 장애 발생 시 어느 시점까지의 데이터를 복원할 수 있는지를 나타냅니다. RPO 24시간이란, 최악의 경우 최대 24시간치 데이터 손실을 감수한다는 의미입니다. **RTO(Recovery Time Objective, 복구 목표 시간)**는 장애 발생 후 정상 서비스까지 걸리는 최대 허용 시간입니다. RTO 4시간이란, 장애 발생 후 4시간 안에 서비스를 재개해야 한다는 목표입니다.

매일 03:00에 자동 백업이 실행되므로, 가장 늦은 시간에 발생한 장애의 RPO는 최대 24시간입니다. 모든 백업 파일은 AES-256으로 암호화되어 저장됩니다.

| 항목 | 규칙 |
|---|---|
| 자동 백업 | 매일 03:00 |
| 보존 | 일간 30일, 월간 1년 |
| 암호화 | AES-256 |
| 복구 요청 | 본사(회장/대표)만 |
| RPO | 24시간 |
| RTO | 4시간 |
