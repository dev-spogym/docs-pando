# 접근 보안 정책

## 비밀번호

| 항목 | 규칙 |
|---|---|
| 최소 길이 | 8자 |
| 복잡도 | 영문+숫자+특수문자 3종 조합 |
| 변경 주기 | 90일 권고, 180일 강제 |
| 재사용 금지 | 최근 3개 |
| 초기 비밀번호 | 최초 로그인 시 강제 변경 |
| 오류 | 5회→30분 잠금, 10회→영구 |

## 세션

| 항목 | 규칙 |
|---|---|
| 비활성 타임아웃 | 30분 |
| 절대 타임아웃 | 12시간 |
| 동시 로그인 | 1계정 1기기 |
| 강제 로그아웃 | 지점장+(직원), 본사(지점장) |

## IP/기기 접근 제한

| 항목 | 규칙 |
|---|---|
| IP 화이트리스트 | 본사 설정 (기본 OFF) |
| 외부 2FA | 본사 설정 |
| 미인가 접근 | 지점장 + 본사 알림 |

## 접근 로그 모니터링

| 패턴 | 대응 |
|---|---|
| 심야 접근 (23~06시) | 지점장 + 본사 알림 |
| 대량 다운로드 (1시간 3회+) | 본사 알림 |
| 빈번 삭제 (1시간 5회+) | 본사 알림 + 차단 |
| 로그인 5회 실패 | 잠금 + 본사 알림 |
