# 감사 로그 정책

import { Callout } from 'nextra/components'

감사 로그는 세 가지 이유에서 필수입니다. 첫째, **법적 의무**입니다. 개인정보보호법은 개인정보 처리 내역을 일정 기간 보존하도록 요구하며, 전자금융거래법은 금융 관련 접근 기록 보존을 의무화합니다. 둘째, **내부 통제**입니다. 누가, 언제, 무엇을 했는지 기록함으로써 직원의 임의 행위를 억제하고 이상 징후를 조기에 발견할 수 있습니다. 셋째, **분쟁 해결**입니다. 회원이 "결제한 적 없다", "탈퇴 요청한 적 없다" 등을 주장할 때, 처리 이력 기록이 유일한 증거 자료가 됩니다.

## 감사 로그 화면

감사 로그 메뉴에서 시스템 전반의 행위 이력을 조회할 수 있습니다.

- **필터**: 기간, 카테고리, 행위자 조건으로 필터링
- **로그 상세**: 각 로그 클릭 → 행위자, 대상, 변경 전/후 데이터, IP, 시간 확인
- **검색**: 특정 회원명이나 행위 유형으로 키워드 검색
- **수기등록 필터**: "수기등록" 태그로 필터링하여 집중 모니터링 가능

## 기록 대상

시스템에서 발생하는 모든 의미 있는 행위가 기록 대상입니다. 특히 결재 이력은 요청·승인·반려 전 건이 기록되어야 합니다. 결재 기록은 누가 어떤 판단을 했는지의 책임 소재를 명확히 하며, 추후 감사나 분쟁 시 핵심 증거가 됩니다.

| 카테고리 | 행위 |
|---|---|
| 인증 | 로그인, 로그아웃, 비밀번호 변경, 지점 전환 |
| 회원 | 등록, 수정, 삭제, 이관, 휴회, 정지, 양도, 블랙리스트 |
| 매출 | 결제, 환불, 수정, 미수금, 수기 등록 |
| 급여 | 등록, 수정, 확정, 지급, 퇴직 정산 |
| 직원 | 등록, 수정, 퇴사, 역할 변경, 지점 이동 |
| 설정 | 센터 설정, 권한, 가격 변경 |
| 시설 | 락커, RFID, 강제 반납 |
| 보안 | 로그인 실패, 잠금, 비정상 접근 |
| **결재** | **요청, 승인, 반려, 보류 전 건** |

## 접근 권한

<Callout type="warning">
감사 로그 자체가 민감한 정보입니다. 로그에는 직원의 모든 행위 기록이 담겨 있으므로, 로그 조회 권한은 관리 책임이 있는 역할에만 부여됩니다. 스태프·FC·트레이너는 감사 로그에 접근할 수 없습니다.
</Callout>

감사 로그 조회 권한을 제한하는 이유는 두 가지입니다. 첫째, 로그에는 동료 직원의 행위 기록이 포함되어 있어 직원 프라이버시 침해가 될 수 있습니다. 둘째, 자신의 행위 기록을 본인이 조회·수정할 수 있다면 감사 로그 자체의 무결성이 무너집니다. 따라서 지점장은 소속 지점 로그만, 본사 역할은 전 지점 로그를 조회할 수 있습니다.

| 역할 | 범위 |
|---|---|
| 회장/대표/본사직원 | 전 지점 전체 |
| 지점장 | 소속 지점만 |
| 그 외 | 접근 불가 |

## 보존 기간

보존 기간은 각 데이터의 법적 의무와 실무적 필요를 반영하여 설정되었습니다.

<Callout type="warning">
금융 관련 로그 5년 보존은 국세기본법상 장부 보존 의무에 근거합니다. 결재 이력 5년 보존은 계약 분쟁 시효(민법 상 일반 채권 소멸시효 5년)를 고려한 것입니다. 개인정보 열람 로그 3년 보존은 개인정보보호법 위반 조사 대비를 위한 기준입니다.
</Callout>

| 카테고리 | 기간 | 주요 근거 |
|---|---|---|
| 일반 행위 | 1년 | 내부 운영 기준 |
| 금융 관련 | 5년 | 국세기본법, 부가가치세법 |
| 개인정보 열람 | 3년 | 개인정보보호법 |
| 보안 이벤트 | 3년 | 개인정보보호법, 정보통신망법 |
| 결재 이력 | 5년 | 민법 소멸시효, 내부 통제 |
