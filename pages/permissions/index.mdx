# 기능별 권한 매트릭스

import { Callout } from 'nextra/components'

권한 매트릭스는 "누가 무엇을 할 수 있는가"를 한눈에 파악할 수 있는 참조 문서입니다. 시스템의 모든 기능에 대해 7개 역할별 허용 여부가 정의되어 있습니다.

## 범례 읽는 방법

<Callout>
**범례:** ✅ 허용 | ❌ 차단 | 🔸 조건부 | 📋 본사 결재 필요

**본사 3역할**은 전 지점에 대해 권한 적용. **지점 4역할**은 소속 지점 한정.
</Callout>

각 기호의 실제 의미를 예시로 설명합니다.

- **✅ 허용**: 해당 역할이 해당 기능을 제한 없이 사용할 수 있습니다. 예: 지점장의 회원 등록은 ✅이므로 언제든 바로 처리 가능합니다.
- **❌ 차단**: 해당 역할은 해당 기능에 접근 자체가 불가합니다. 메뉴가 보이지 않거나, 버튼이 비활성화됩니다. 예: 트레이너는 POS 결제가 ❌이므로 결제 화면 자체가 표시되지 않습니다.
- **🔸 조건부**: 특정 조건 하에서만 허용됩니다. 예: 트레이너의 수업 등록은 🔸로, **본인 수업만** 등록 가능합니다. FC의 강사 현황도 🔸로, **본인 실적만** 조회됩니다.
- **📋 본사 결재 필요**: 해당 역할이 해당 행위를 하려면 먼저 본사에 결재를 올리고 승인을 받아야 합니다. 예: 지점장의 정규직 채용은 📋이므로, 결재 승인 전에는 시스템에서 처리가 불가합니다. 승인 후에는 ✅처럼 처리 가능해집니다.

## 본사 역할 vs 지점 역할

**본사 3역할(회장·대표·본사직원)**은 테넌트 내 모든 지점의 데이터에 접근할 수 있습니다. 강남점 데이터든 역삼점 데이터든 구분 없이 조회·처리가 가능합니다.

**지점 4역할(지점장·스태프·FC·트레이너)**은 자신이 소속된 지점의 데이터만 접근할 수 있습니다. 강남점 소속 지점장은 역삼점 회원 목록을 볼 수 없고, 역삼점 매출 통계에도 접근할 수 없습니다. 이는 지점 간 데이터 격리를 통해 개인정보 보호와 운영 보안을 확보하기 위한 원칙입니다.
