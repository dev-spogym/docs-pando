# 지점 구조와 데이터 격리

import { Callout } from 'nextra/components'

## 멀티테넌트 구조

판도 CRM은 멀티테넌트 아키텍처로 설계되어 있습니다. 테넌트(Tenant)란 하나의 법인 또는 브랜드 단위를 의미합니다. 예를 들어 "판도 피트니스"라는 브랜드가 하나의 테넌트이고, 그 아래 강남점·역삼점·판교점이 각각 지점으로 속합니다. 서로 다른 테넌트(다른 법인)의 데이터는 완전히 격리되어 어떤 상황에서도 교차 접근이 불가능합니다.

```
Tenant (법인/브랜드)
  ├── 본사 (HQ)
  │     ├── 회장
  │     ├── 대표 ─── 겸임 → 강남점, 역삼점
  │     └── 본사 직원 A, B, C
  │
  ├── 지점 A (강남점) ─ ACTIVE
  │     ├── 지점장
  │     ├── 스태프 × 2
  │     ├── FC × 2
  │     └── 트레이너 × 3
  │
  ├── 지점 B (역삼점) ─ ACTIVE
  └── 지점 C (판교점) ─ PAUSED
```

## 지점 상태 (4종)

지점 상태는 해당 지점에서 허용되는 행위의 범위를 결정합니다. 일시중단(PAUSED)과 폐점진행(CLOSING) 상태로 전환하려면 회장 승인이 필요하며, 이는 전사 운영에 미치는 영향이 크기 때문입니다.

| 상태 | 코드 | 설명 |
|---|---|---|
| 운영중 | `ACTIVE` | 정상 운영. 모든 기능 사용 가능 |
| 일시중단 | `PAUSED` | 임시 휴점. 회원 출입/예약 차단. 데이터 조회만 가능 |
| 폐점진행 | `CLOSING` | 폐점 절차 중. 환불·이관 처리 진행. 신규 등록 차단 |
| 폐점완료 | `INACTIVE` | 아카이브. 조회 전용. 데이터 보존 |

**상태별 허용/차단 행위:**

- **ACTIVE**: 신규 회원 등록, POS 결제, 수업 예약, 출입, 모든 기능 정상 사용
- **PAUSED**: 회원 출입·예약 차단. 기존 데이터 조회 및 환불 처리는 허용. 신규 판매 불가
- **CLOSING**: 신규 회원 등록·상품 판매 불가. 기존 회원 환불·타 지점 이관 처리에 집중
- **INACTIVE**: 읽기 전용. 과거 데이터 조회와 세무·감사 목적의 기록 열람만 허용

## 데이터 격리 원칙

<Callout type="warning">
데이터 격리는 개인정보 보호와 경영 기밀 보호 모두를 위한 핵심 원칙입니다. 강남점 트레이너가 역삼점 회원의 연락처나 결제 내역에 접근하는 것은 개인정보보호법 위반이 될 수 있습니다.
</Callout>

| 원칙 | 설명 |
|---|---|
| **본사 → 전 지점** | 본사 역할은 전 지점 CRUD 가능 |
| **지점 → 소속만** | 지점 역할은 소속 지점만 접근 |
| **지점 ↔ 타 지점** | 교차 접근 불가 (이관 등 예외) |
| **테넌트 격리** | 다른 법인 데이터 접근 불가 |

지점 간 데이터 교차 접근이 허용되는 유일한 예외는 **회원 이관** 처리입니다. 이 경우에도 이관 처리에 필요한 최소한의 데이터만 공유되며, 수신 지점 지점장의 승인이 필수입니다.

## 사용자 소속 규칙

사용자의 소속은 `branchId` 필드로 관리됩니다. 본사 소속 사용자는 `branchId`가 null이며 전 지점 전환이 가능합니다. 지점 소속 사용자는 특정 `branchId`에 고정되어 지점 전환이 불가능합니다.

| 역할 | 소속 | 지점 전환 |
|---|---|---|
| 회장 | 본사 (`branchId` null) | 전 지점 전환 가능 |
| 대표 | 본사 + 겸임 지점 | 전 지점 전환 가능 |
| 본사 직원 | 본사 (`branchId` null) | 전 지점 전환 가능 |
| 지점장 | 특정 지점 1개 | 불가 |
| 스태프/FC/트레이너 | 특정 지점 1개 | 불가 |

**대표의 겸임 구조가 실무에서 작동하는 방식:**

대표가 강남점과 역삼점을 겸임하는 경우, 시스템 상에서 두 지점의 지점장 권한을 동시에 갖습니다. 대표는 본사 대시보드에서 전 지점 통합 현황을 보다가, 강남점 뷰로 전환하여 해당 지점의 일일 매출이나 회원 현황을 지점장 수준에서 직접 관리할 수 있습니다. 이를 통해 별도 지점장 없이 소규모 지점을 효율적으로 운영할 수 있습니다.

## 지점 관리 화면 사용법

### 지점 목록 확인

본사 역할로 로그인한 후 지점 관리 메뉴로 이동하면 전체 지점 목록이 카드 형태로 표시됩니다. 각 카드에서 해당 지점의 현재 상태, 회원 수, 매출, 직원 수 요약을 바로 확인할 수 있습니다.

### 지점 등록

지점 목록 화면에서 "새 지점" 버튼을 누릅니다. 지점 이름, 지점 코드, 주소, 연락처 등 기본 정보를 입력하고 저장하면 새 지점이 등록됩니다.

### 지점 비활성화

지점 상세 화면에서 "비활성화" 버튼을 누르면 비활성화 요청이 생성됩니다. 비활성화는 회장 승인이 필요하며, 실제 데이터는 삭제되지 않고 `INACTIVE` 상태로 보존됩니다.

<Callout type="warning">
지점을 비활성화하기 전에 소속 회원의 환불 또는 타 지점 이관 처리를 먼저 완료하세요. 비활성화 후에는 신규 등록과 결제가 차단됩니다.
</Callout>
