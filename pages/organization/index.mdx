# 조직 구조

import { Callout } from 'nextra/components'

판도 CRM은 **본사-지점 2계층 구조**로 운영됩니다. 이 구조는 여러 지점을 운영하는 피트니스 센터 기업의 현실을 반영한 설계입니다. 본사는 전사적 정책 수립과 경영 통제를 담당하고, 지점은 실제 회원 서비스와 일상 운영을 담당합니다. 두 계층의 역할을 명확히 분리함으로써 지점의 운영 자율성을 보장하면서도 본사가 전체 경영 방향을 일관되게 유지할 수 있습니다.

```
본사 (HQ)
  ├── 회장
  ├── 대표 ─── 겸임 → 강남점, 역삼점
  └── 본사 직원 A, B, C
       │
       ├── 지점 A (강남점) ─ ACTIVE
       │     ├── 지점장
       │     ├── 스태프 × 2
       │     ├── FC × 2
       │     └── 트레이너 × 3
       │
       ├── 지점 B (역삼점) ─ ACTIVE
       └── 지점 C (판교점) ─ PAUSED
```

## 계층별 역할

본사는 전 지점에 걸친 데이터 접근 및 경영 의사결정 권한을 갖습니다. 반면 지점 역할은 소속 지점의 데이터에만 접근할 수 있으며, 경영·재무·정책에 영향을 미치는 행위는 반드시 본사 결재를 거쳐야 합니다.

| 계층 | 역할 | 수 |
|---|---|---|
| **본사** | 회장, 대표, 본사 직원 | 3종 |
| **지점** | 지점장, 스태프, FC, 트레이너 | 4종 |

## 핵심 원칙

- 본사 역할은 **전 지점 데이터** CRUD 가능
- 지점 역할은 **소속 지점만** 접근
- 대표는 본사 소속이면서 **특정 지점 겸임** 가능
- 지점에서 경영/재무/정책 관련 행위 시 **본사 결재 필수**

## 데이터 흐름과 보고 체계

<Callout type="info">
지점의 모든 매출·회원·운영 데이터는 실시간으로 본사에서 조회 가능합니다. 지점장은 이상 상황(미수금 급증, 회원 이탈 등) 발생 시 본사에 보고하고 결재를 요청해야 합니다.
</Callout>

지점에서 생성된 모든 데이터는 테넌트 단위로 격리되어 저장됩니다. 본사 역할(회장·대표·본사 직원)은 소속 테넌트 내 전 지점 데이터에 자유롭게 접근할 수 있지만, 지점 역할은 자신이 소속된 지점의 데이터만 볼 수 있습니다. 다른 지점의 회원 정보나 매출 데이터는 이관 처리 등 명시적으로 허용된 예외 상황을 제외하고는 접근이 차단됩니다.

대표의 겸임 구조는 소규모 법인에서 대표가 직접 특정 지점을 운영하는 현실을 반영합니다. 겸임 대표는 본사 권한과 해당 지점의 지점장 권한을 동시에 행사할 수 있어, 별도 지점장 없이도 지점을 직접 관리할 수 있습니다.

## 조직 관리 화면 사용법

### 조직도 확인

설정 메뉴 → 조직 관리에서 본사와 지점의 전체 구조를 확인할 수 있습니다. 각 지점별 소속 직원 목록과 역할을 한눈에 파악할 수 있습니다.

### 지점 전환 (본사 역할)

본사 역할(회장, 대표, 본사 직원)로 로그인하면 화면 상단 헤더에 지점 선택 드롭다운이 표시됩니다. 드롭다운에서 원하는 지점을 선택하면 해당 지점의 뷰로 전환되어 지점 데이터를 직접 확인하고 관리할 수 있습니다.
