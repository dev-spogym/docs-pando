# 조직 구조

판도 CRM은 **본사-지점 2계층 구조**로 운영됩니다.

```
본사 (HQ)
  ├── 회장
  ├── 대표 ─── 겸임 → 강남점, 역삼점
  └── 본사 직원 A, B, C
       │
       ├── 지점 A (강남점) ─ ACTIVE
       │     ├── 지점장
       │     ├── 스태프 × 2
       │     ├── FC × 2
       │     └── 트레이너 × 3
       │
       ├── 지점 B (역삼점) ─ ACTIVE
       └── 지점 C (판교점) ─ PAUSED
```

## 계층별 역할

| 계층 | 역할 | 수 |
|---|---|---|
| **본사** | 회장, 대표, 본사 직원 | 3종 |
| **지점** | 지점장, 스태프, FC, 트레이너 | 4종 |

## 핵심 원칙

- 본사 역할은 **전 지점 데이터** CRUD 가능
- 지점 역할은 **소속 지점만** 접근
- 대표는 본사 소속이면서 **특정 지점 겸임** 가능
- 지점에서 경영/재무/정책 관련 행위 시 **본사 결재 필수**
