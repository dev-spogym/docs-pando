# 매출 마감 정책

import { Callout } from 'nextra/components'

매출 마감은 센터의 재무 데이터를 확정하는 핵심 프로세스입니다. 일 마감은 당일 거래 내역을 스냅샷으로 저장하여 이후 수정 추적을 가능하게 하고, 월 마감은 해당 월의 모든 수치를 확정하여 급여 정산, 세금 신고, 경영 보고의 기준이 됩니다. 마감 데이터의 무결성을 보호하기 위해 마감 이후의 수정은 권한을 엄격히 제한합니다.

## 일 마감

일 마감은 매일 영업 종료 시 시스템이 자동으로 처리합니다. 당일의 POS 결제, 환불, 수기 등록 내역을 확정하고 일별 매출 스냅샷을 생성합니다. 지점장은 영업 종료 전 당일 거래 내역의 이상 여부를 반드시 확인해야 합니다.

**일 마감과 월 마감의 차이:**
- 일 마감은 운영 차원의 마감으로, 당일 수정은 지점장이 가능하고 전일 이전 수정만 본사 결재가 필요합니다.
- 월 마감은 회계 차원의 확정으로, 일단 확정되면 회장만 수정할 수 있으며 수정 사유가 AuditLog에 기록됩니다.

| 항목 | 규칙 |
|---|---|
| 마감 시간 | 영업 종료 시 (기본 23:00) |
| 마감 처리 | 시스템 자동 (일별 매출 snapshot) |
| 당일 수정 | 지점장 |
| 전일 수정 | 본사 결재 |
| 리포트 | 지점장 + 본사 자동 발송 |

## 월 마감

월 마감은 매월 말일 23:59에 자동으로 진행되며, 본사(대표/회장)가 익월 5일까지 최종 확정합니다. 확정 전에는 이상 거래, 미수금, 수기 등록 내역 등을 검토하여 오류를 바로잡을 수 있습니다.

**월 마감 확정 후 수정이 회장만 가능한 이유:**
월 마감이 확정되면 해당 월의 매출 데이터는 급여 인센티브 산정, 부가세 신고, 경영 보고의 기준값이 됩니다. 이 시점 이후에 데이터를 수정하면 이미 산출된 급여·세금과의 정합성이 깨질 수 있습니다. 따라서 최고 의사결정권자인 회장만이 사유를 명시하고 수정할 수 있도록 제한하여, 회계 데이터의 신뢰성과 감사 가능성을 보장합니다.

<Callout type="warning">
월 마감 확정 후 수정은 **회장 전용**입니다. 대표, 본사 직원, 지점장은 수정 불가합니다. 수정이 필요한 경우 회장에게 사유를 보고하고 요청해야 합니다.
</Callout>

| 항목 | 규칙 |
|---|---|
| 마감일 | 매월 말일 23:59 (자동) |
| 확정 | 본사(대표/회장) — 익월 5일까지 |
| 확정 후 수정 | 회장만 |
| 마감 전 알림 | 말일 3일 전 지점장 알림 |
| 리포트 | 본사 + 각 지점장 자동 배포 |

## 수기 등록 매출

수기 등록은 POS 시스템 장애, 현장 현금 거래, 외부 결제(타 플랫폼 등) 등 예외적인 상황에서 지점장이 직접 매출을 시스템에 입력하는 기능입니다. 일반적인 POS 결제와 달리 시스템 검증을 거치지 않기 때문에 오입력·누락·허위 등록의 위험이 있습니다.

**수기 등록이 감사 강화 대상인 이유:**
수기 등록은 카드 단말기나 PG사 같은 외부 기관의 검증 없이 지점장 단독으로 처리되는 거래입니다. 따라서 등록 즉시 본사(대표)에게 실시간 알림이 발송되고, AuditLog에 "수기등록" 태그가 별도 표시됩니다. 이를 통해 본사는 수기 등록 건을 집중 모니터링하여 회계 투명성을 확보합니다. 지점장은 수기 등록 후 원본 영수증 또는 입금 확인서를 반드시 보관해야 합니다.

**월 마감 리포트 포함 항목:**
- 총 매출(POS 결제 + 수기 등록), 결제 수단별 매출 내역
- 환불 건수 및 금액, 위약금 수입
- 미수금 현황, 신규 등록/재등록 회원 수
- 수기 등록 건수 및 금액 (별도 항목으로 표시)
- 전월 대비 매출 증감률

<Callout type="warning">
수기 등록 매출은 등록 즉시 본사 대표에게 **실시간 알림**이 발송됩니다. 임의 누락이나 허위 등록은 감사 대상이 될 수 있습니다.
</Callout>

| 항목 | 규칙 |
|---|---|
| 사유 | POS 장애, 현금 직거래, 외부 결제 |
| 권한 | 지점장 |
| 감사 | 본사(대표)에게 실시간 알림 |
| 표시 | AuditLog에 "수기등록" 태그 |

## CRM에서 마감 처리하기

### 일 마감 확인

매출 → 마감 메뉴에서 당일 거래 내역을 확인합니다. 이상이 없으면 시스템이 영업 종료 시각에 자동으로 마감 처리합니다. 당일 수정이 필요한 경우 지점장이 직접 수정할 수 있으며, 전일 이전 거래 수정은 본사 결재가 필요합니다.

### 월 마감 확인

매출 → 월 마감에서 월간 리포트를 검토합니다. 이상 거래, 미수금, 수기 등록 내역을 확인한 뒤 본사 확정을 대기합니다. 본사(대표/회장)가 익월 5일까지 최종 확정하며, 확정 이후 수정은 회장만 가능합니다.

### 수기 등록

매출 → "수기 등록" 버튼을 클릭하면 결제 정보를 직접 입력할 수 있습니다. 등록 시 사유 입력이 필수이며, 등록 즉시 본사 대표에게 실시간 알림이 발송됩니다. 원본 영수증 또는 입금 확인서는 반드시 보관해야 합니다.

### 마감 수정 권한

| 시점 | 수정 권한 |
|---|---|
| 확정 전 (당일) | 지점장 |
| 확정 전 (전일 이전) | 본사 결재 |
| 월 마감 확정 후 | 회장만 |
