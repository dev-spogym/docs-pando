# 회원 관리

import { Callout } from 'nextra/components'

회원 상태는 단순한 분류 체계가 아니라 실제 서비스 이용 가능 여부를 결정하는 핵심 메커니즘입니다.

## 회원 관리 기능 개요

CRM에서 회원 관리는 목록 조회부터 등록, 상세 확인, 일괄 작업까지 한 곳에서 처리할 수 있습니다.

### 회원 목록 조회

상단 메뉴에서 **회원 관리**를 선택하면 전체 회원 목록이 표시됩니다.

- **검색**: 이름, 연락처로 빠르게 찾을 수 있습니다.
- **필터**: 상태(ACTIVE, HOLDING, SUSPENDED 등), 이용권 유형별로 좁혀서 볼 수 있습니다.
- **페이지네이션**: 목록이 많을 경우 하단 페이지 버튼으로 이동합니다.
- 회원 행을 클릭하면 해당 회원의 상세 화면으로 이동합니다.

### 회원 등록

목록 화면 우측 상단의 **회원 등록** 버튼을 클릭합니다.

1. **기본정보**: 이름, 연락처, 생년월일, 성별을 입력합니다.
2. **주소**: 선택 항목으로 입력할 수 있습니다.
3. **메모**: 상담 내용이나 특이사항을 자유롭게 기록합니다.
4. **저장** 버튼을 누르면 즉시 ACTIVE 상태로 등록됩니다.

<Callout type="info">
미성년자(만 14세 미만)는 등록 시 **보호자 정보(이름, 연락처, 동의서)** 입력이 필수입니다. 입력하지 않으면 저장이 되지 않습니다.
</Callout>

### 회원 수정

회원 상세 화면 우측 상단의 **수정** 버튼을 클릭합니다.

- 이름, 연락처, 생년월일 등 기본정보를 변경할 수 있습니다.
- **상태 변경** 드롭다운에서 HOLDING, SUSPENDED 등으로 직접 전환할 수 있습니다(권한 있는 직원만).
- **메모 추가** 탭에서 상담 이력을 기록할 수 있습니다.

### 회원 상세 화면 탭 구성

회원을 클릭하면 아래 탭으로 구성된 상세 화면이 열립니다.

| 탭 | 내용 |
|---|---|
| 기본정보 | 연락처, 생년월일, 상태, 등록일 등 |
| 체형정보 | 체성분 측정 이력 |
| 상담이력 | FC/스태프 상담 기록 |
| 결제상세 | 이용권 구매 및 환불 내역 |
| 예약현황 | 수업 예약 목록 및 출석 현황 |
| 운동로그 | 운동 기록 |
| 운동프로그램 | 배정된 운동 프로그램 |
| 평가 | 체력 평가 기록 |

### 회원 일괄 작업

목록 화면에서 체크박스로 여러 회원을 선택한 후 상단에 나타나는 **일괄 작업** 메뉴를 사용합니다.

- **엑셀 다운로드**: 선택한 회원 정보를 엑셀 파일로 내려받습니다.
- **일괄 SMS 발송**: 선택한 회원에게 동일한 문자를 한 번에 보낼 수 있습니다.

## 회원 상태 전이 규칙 회원의 상태에 따라 출입 게이트 통과, 수업 예약, 결제, 환불 가능 여부가 모두 달라집니다. 상태 관리가 정확하지 않으면 만료된 회원이 시설을 계속 이용하거나, 정지된 회원이 예약을 잡는 문제가 발생합니다. 각 상태 전이는 시스템 자동 처리 또는 권한 있는 직원의 수동 처리로만 가능합니다.

## 전체 상태 흐름도

```
[신규 등록] → ACTIVE
                 │
        ┌────────┼────────┬──────────┬──────────┐
        ↓        ↓        ↓          ↓          ↓
    HOLDING   SUSPENDED  EXPIRED  WITHDRAWN  TRANSFERRED
     (휴회)     (정지)    (만료)     (탈퇴)      (이관)
        │        │        │
        ↓        ↓        ↓
     ACTIVE   ACTIVE   DORMANT ──→ [데이터 파기]
    (복귀)    (해제)    (휴면)       (3년 후)
```

## 상태별 허용 행위

각 상태에서 허용되는 행위는 해당 상태의 비즈니스 목적에 맞게 설계되어 있습니다.

- **ACTIVE**: 모든 서비스를 정상 이용할 수 있는 상태입니다.
- **HOLDING**: 휴회 중으로 시설 이용은 차단되지만 환불은 가능합니다. 이관도 허용하여 휴회 중에도 타 지점으로 옮길 수 있습니다.
- **SUSPENDED**: 정지 상태에서는 시설 이용이 전면 차단됩니다. 단, 미납으로 인한 정지의 경우 미납금 납부(결제)는 허용합니다. 미납을 해소해야 정지가 해제되기 때문입니다.
- **EXPIRED**: 이용권이 만료되었지만 아직 휴면 전환 전입니다. 재등록 결제가 가능하고, 환불도 처리할 수 있습니다.
- **DORMANT**: 휴면 상태로 전환된 회원입니다. 재등록을 통해 즉시 ACTIVE로 복귀할 수 있지만, 데이터 보호 관점에서 일반 결제는 차단됩니다.

<Callout type="info">
SUSPENDED 상태에서 결제(🔸)는 **미납 해소 목적으로만** 허용됩니다. 새로운 이용권 구매나 PT 패키지 결제는 정지 해제 후에 가능합니다.
</Callout>

| 행위 | ACTIVE | HOLDING | SUSPENDED | EXPIRED | DORMANT | WITHDRAWN |
|---|:---:|:---:|:---:|:---:|:---:|:---:|
| 출입 | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ |
| 수업 예약 | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ |
| 결제 | ✅ | ❌ | 🔸 | ✅ | ✅ | ❌ |
| 환불 | ✅ | ✅ | ✅ | ✅ | ❌ | ❌ |
| 휴회 신청 | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ |
| 양도 | ✅ | ❌ | ❌ | ❌ | ❌ | ❌ |
| 이관 | ✅ | ✅ | ❌ | ✅ | ❌ | ❌ |

- SUSPENDED 결제: 미납 해소 목적만

## 상태 전이 트리거

상태 전이는 크게 두 가지 방식으로 발생합니다. **시스템 자동 처리**는 매일 자정(00:00) 배치 작업 또는 특정 조건 충족 즉시 실행됩니다. **수동 처리**는 권한이 있는 직원이 직접 시스템에서 상태를 변경합니다.

| 전이 | 트리거 | 자동/수동 | 수행 권한 |
|---|---|---|---|
| → ACTIVE | 신규 등록, 재등록, 복귀, 해제 | 수동/자동 | 스태프+ / 시스템 |
| ACTIVE → HOLDING | 휴회 신청 | 수동 | 스태프+ |
| HOLDING → ACTIVE | 종료일 도래 / 조기 복귀 | 자동/수동 | 시스템 / 스태프+ |
| ACTIVE → SUSPENDED | 미납 30일+ / 규정 위반 | 자동/수동 | 시스템 / 지점장 |
| SUSPENDED → ACTIVE | 미납 해소 / 기간 만료 | 자동/수동 | 시스템 / 지점장 |
| ACTIVE → EXPIRED | 이용 기간/횟수 소진 | 자동 | 시스템 |
| EXPIRED → DORMANT | 만료 후 90일 경과 | 자동 | 시스템 |
| ACTIVE → WITHDRAWN | 탈퇴 요청 | 수동 | 지점장+ |
| ACTIVE → TRANSFERRED | 이관 완료 | 수동 | 지점장+ |
