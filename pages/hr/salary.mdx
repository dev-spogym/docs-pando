# 급여 체계 / 인센티브

import { Callout } from 'nextra/components'

급여 체계는 직원의 역할과 고용 형태에 따라 다르게 설계되어 있습니다. 각 유형은 해당 역할의 업무 특성과 동기부여 구조를 반영합니다. 급여 체계는 본사(회장/대표)가 결정하며, 변경 시 반드시 본사 결재를 받아야 합니다.

## 급여 유형

각 급여 유형이 적합한 대상과 이유는 다음과 같습니다.

- **월급제 (지점장, 스태프):** 매일 정해진 업무(회원 응대, 시설 관리, 매출 관리 등)를 수행하는 역할에 적합합니다. 업무량이 예측 가능하고 안정적이므로 고정 기본급이 적합하며, 직책수당은 책임의 대가입니다.
- **시급제 (파트타임):** 출퇴근 시간이 불규칙하고 단기 계약인 파트타임 직원에게 적합합니다. 실제 근무한 시간만큼만 지급하므로 센터 입장에서 유연한 인력 운용이 가능합니다.
- **건별제 (프리랜서 트레이너):** 고용 계약 없이 수업 단위로 계약하는 외부 트레이너에게 적합합니다. 수업 1건당 단가를 지급하므로, 수업이 없는 날은 비용이 발생하지 않습니다.
- **혼합제 (정규 FC, 트레이너):** 기본적인 생활 안정을 위한 기본급과 성과에 따른 인센티브를 결합한 구조입니다. 안정감을 주면서도 높은 성과에 대한 강력한 동기부여를 제공합니다.

| 유형 | 대상 | 구성 |
|---|---|---|
| 월급제 | 지점장, 스태프 | 기본급 + 직책수당 + 인센티브 - 공제 |
| 시급제 | 파트타임 | 시급 × 근무시간 + 인센티브 - 공제 |
| 건별제 | 트레이너 (프리랜서) | 건당 단가 + 인센티브 - 공제 |
| 혼합제 | FC, 트레이너 (정규) | 기본급 + 성과 수당 + 인센티브 - 공제 |

급여 체계 결정: **본사(회장/대표)**, 변경 시 본사 결재

## PT 수업 인센티브 (트레이너)

PT 인센티브는 매출 구간이 높아질수록 인센티브율이 올라가는 누진 구조입니다. 이 구조는 트레이너가 더 많은 수업을 진행할수록 더 큰 보상을 받도록 설계되어, 적극적인 회원 유치와 수업 품질 향상을 동시에 유도합니다.

**인센티브 계산 예시:**
- 월 PT 매출 200만 원: 200만 원 × 30% = **60만 원**
- 월 PT 매출 400만 원: 400만 원 × 35% = **140만 원**
- 월 PT 매출 650만 원: 650만 원 × 40% = **260만 원**
- 월 PT 매출 1,000만 원: 1,000만 원 × 45% = **450만 원**

```
PT 인센티브 = PT 매출 × 인센티브율

인센티브율:
  월 300만 원 이하 → 30%
  월 300~500만 원 → 35%
  월 500~800만 원 → 40%
  월 800만 원 초과 → 45%
```

## GX 수업 인센티브 (트레이너)

GX 인센티브는 수업당 참여 인원에 따라 단가가 달라지는 구조입니다. 참여 인원이 많을수록 단가가 높아지므로, 트레이너가 회원들의 참여를 적극적으로 유도하는 동기를 제공합니다. 동시에 인기 있는 수업을 진행하는 트레이너가 더 높은 보상을 받는 공정한 구조이기도 합니다.

```
GX 인센티브 = 수업 횟수 × 회당 단가

회당 단가:
  5인 이하 → 20,000원
  6~10인 → 25,000원
  11~15인 → 30,000원
  16인+ → 35,000원
```

## FC 인센티브

FC 인센티브는 트레이너 인센티브와 구조적으로 다릅니다. 트레이너는 수업 매출에 비례한 인센티브를 받는 반면, FC는 판매 행위 자체(신규 유치, 재등록 유도)에 따른 건별 수수료와 매출 비례 수수료를 혼합하여 받습니다. FC의 핵심 역할이 "영업"이기 때문에, 회원을 데려오고 재계약을 이끌어내는 행위에 직접적인 보상이 연결됩니다.

**FC vs 트레이너 인센티브 구조 비교:**
- 트레이너: 수업 1회 진행 → 매출 발생 → 비율 인센티브 (수업 제공의 대가)
- FC: 상담 → 계약 전환 → 건별 수수료 + 판매액의 일정 비율 (영업 성과의 대가)

```
판매 인센티브 = 본인 판매 매출 × 수수료율

수수료율:
  이용권 판매 → 8~15%
  재등록 유도 → 건당 20,000원
  신규 유치 (상담→계약) → 건당 30,000원
```

## 인센티브 관리

인센티브 시뮬레이션 기능은 직원이 현재 실적을 바탕으로 이번 달 예상 인센티브를 미리 확인할 수 있는 도구입니다. 이를 통해 직원은 목표 달성까지 얼마나 더 노력해야 하는지를 직관적으로 파악하고, 자발적인 동기부여로 이어집니다. 예를 들어 트레이너가 현재 PT 매출이 280만 원인 경우, 300만 원 구간 돌파 시 인센티브율이 30%→35%로 높아지는 것을 확인하고 추가 수업을 유도하는 식입니다.

<Callout type="info">
인센티브 구간 및 수수료율은 본사가 설정하며, 변경 시 지점장이 본사 결재를 받아야 합니다. 직원은 본인 인센티브 내역을 언제든 시스템에서 조회할 수 있습니다.
</Callout>

| 행위 | 권한 |
|---|---|
| 구간/율 설정 | 본사, 지점장 → 결재 |
| 전 직원 조회 | 본사, 지점장 |
| 본인 조회 | 전 역할 |
| 정산 주기 | 매월 (익월 급여 합산) |

## CRM에서 급여 관리하기

### 급여 관리 화면

CRM 좌측 메뉴에서 **급여 관리**를 선택하면 해당 월 전 직원의 급여 테이블을 확인할 수 있습니다. 테이블에는 직원별로 기본급, 인센티브, 공제 항목, 실지급액이 표시됩니다. 상단 월 선택 필터로 과거 월 데이터도 조회할 수 있습니다.

### 급여 등록 및 수정

본사 담당자는 급여 관리 화면에서 직원을 선택한 후 기본급·직책수당·추가 공제 등 항목별 금액을 직접 입력하거나 수정할 수 있습니다. 수정 후 저장하면 실지급액이 자동으로 재계산됩니다.

### 인센티브 자동 계산

시스템이 매월 PT·GX·FC 매출 실적을 기반으로 각 직원의 인센티브를 자동으로 산출합니다. 별도로 수동 입력할 필요 없이 확정된 매출 데이터를 바탕으로 인센티브 구간이 적용됩니다.

### 인센티브 시뮬레이션

직원 본인은 CRM 급여 메뉴에서 **시뮬레이션** 버튼을 클릭하여 현재까지의 실적을 바탕으로 이번 달 예상 인센티브를 미리 확인할 수 있습니다. 매출 구간 경계에 얼마나 근접했는지도 함께 표시되므로, 추가 목표 설정에 활용할 수 있습니다.

<Callout type="info">
인센티브 시뮬레이션 결과는 현재 시점 기준 예상값이며, 월말 최종 정산 시 실적에 따라 달라질 수 있습니다.
</Callout>

### 급여 확정 프로세스

급여는 아래 단계를 순서대로 거쳐 최종 지급됩니다.

| 단계 | 상태 | 처리자 |
|---|---|---|
| 1단계 | DRAFT | 시스템 자동 산출 (매월 1일) |
| 2단계 | PENDING | 지점장 검토 및 이상 확인 |
| 3단계 | PENDING → 검토 | 본사 검토 및 수정 |
| 4단계 | CONFIRMED | 대표/회장 최종 확정 |
| 5단계 | PAID | 지급 완료 및 명세서 발송 |

### 급여 요약

급여 관리 화면 하단에는 해당 월 전 직원의 기본급 합계, 인센티브 합계, 공제 합계, 실지급액 합계 및 평균이 자동으로 계산되어 표시됩니다.
